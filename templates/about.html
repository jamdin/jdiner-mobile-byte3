Accountability: About This Data Set

Dataset Overview

The dataset used for this analysis was collected from my personal mobile phone using the AWARE Framework. The start date for the data is 01-27-2017, we don't have an end date as the data is still being generated, which means that the graphs generated are all up to date. The dataset records data into 7 different tables:

  • Accelerometer: records measures from the accelerometer sensor in my phone.
  • Linear_Accelerometer: records measures from the accelerometer sensor in my phone.
  • Locations: records the longitude and latitude along with the timestamp using the GPS sensor.
  • Locations_Visit: similar to Locations but also gives the street address of the coordinates.
  • Plugin_google_activity_recognition: records the activity that I'm performing: Walking, Running, Still, and In-Vehicle.
  • Plugin_ios_activity_recognition: similar to the previous table but generated by the iPhone.
  • Plugin_openweather: records the weather for the location where I'm at. The refresh rate is once per hour.
Out of these 7 tables, I ended up using only the Locations_Visit, Plugin_gogle_activity_recognition, and the Plugin_OpenWeather.

Where did this data come from?

The dataset was collected on my mobile phone using the AWARE framework, which is automatically uploaded to Google Cloud SQL at least once per day. AWARE uses the sensors in my phone to record the data.

Who has access to it?

The access is very restricted. Only this website, which is connected to the cloud platform, the AWARE platform, and myself from a couple authorized IP addresses (defined on the Cloud SQL authorized users) are able to access the data. Potential unauthorized access is minimal considering Google Privacy Policies.
